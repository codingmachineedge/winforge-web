## Catalog UI/UX & Navigation

1. **Collapsible Sidebar Sections** — Sidebar section headers expand/collapse to hide their module groups, with per-section state persisted. _Feasibility:_ Local component state with a `Map<sectionId, boolean>`, CSS `max-height` transition for the reveal, persisted to `localStorage`.
2. **Grid/List View Toggle** — A toolbar switch flips the module area between card-grid and dense single-column list layouts. _Feasibility:_ One `viewMode` state driving a CSS class on the container; grid uses `display:grid`, list uses `display:flex column`; trivial in plain React/CSS.
3. **Breadcrumb Trail** — Section > Group > Module breadcrumb above the content pane, each segment clickable to jump back. _Feasibility:_ Derive segments from the current selection state array; render `<button>`s that set navigation state — no router needed since navigation is state-driven.
4. **Pinned Favorites Rail** — Users pin modules to a persistent top-of-sidebar rail for one-click access. _Feasibility:_ `Set<moduleId>` in state persisted to `localStorage`; pin toggle on each card; render pinned set above the section tree.
5. **Recently Viewed Strip** — A horizontal strip of the last N opened modules shown on the home/landing view. _Feasibility:_ Maintain an LRU array (max 12) in `localStorage`, push on module open, render as a scrollable flex row.
6. **Drag-to-Reorder Pins** — Reorder pinned modules by dragging within the favorites rail. _Feasibility:_ HTML5 Drag and Drop API (`draggable`, `onDragStart/onDragOver/onDrop`) reordering the persisted array; no library required.
7. **Master-Detail Split Pane** — Resizable split with module list on the left, live detail on the right instead of full-page navigation. _Feasibility:_ CSS grid with a draggable divider updating a `--split-width` CSS variable via pointer events; standard resizer pattern.
8. **Card Density Setting** — Compact / Comfortable / Spacious density presets change card padding and grid gap globally. _Feasibility:_ A `data-density` attribute on the root; CSS variables (`--card-pad`, `--grid-gap`) scoped per density value; persisted setting.
9. **Skeleton Loading Cards** — Animated shimmer placeholders while catalog data or module details load. _Feasibility:_ CSS `@keyframes` shimmer on gray placeholder divs matching card dimensions; render conditionally on a `loading` boolean.
10. **Empty-State Illustrations** — Friendly illustrated empty states for no-results, empty-pins, and empty-recent views. _Feasibility:_ Inline SVG components rendered when a list length is 0, with a call-to-action button; pure React/SVG.
11. **Module Card Hover Preview** — Hovering a card reveals a tooltip/popover with description and quick actions without opening detail. _Feasibility:_ CSS `:hover` + a positioned popover using `getBoundingClientRect`; delay via `setTimeout` in a hover handler.
12. **Sticky Section Headers** — Section headers stick to the top of the scroll container as you scroll their group. _Feasibility:_ CSS `position: sticky; top: 0` on header elements inside the scroll container; zero JS.
13. **Back/Forward Navigation Stack** — In-app back and forward buttons that traverse a history of viewed modules/sections. _Feasibility:_ A history array plus pointer index in state; push on navigation, splice on branch; buttons enabled by index bounds.
14. **Command-Palette Launcher** — Ctrl-style quick-jump overlay to fuzzy-navigate to any module (navigation only, not search algorithm). _Feasibility:_ A modal overlay with an input filtering module titles by substring; renders results as navigable list; opens via a launcher button/shortcut hook owned here as UI shell.
15. **Tabbed Multi-Module View** — Open multiple modules as browser-style tabs across the top for side-by-side workflows. _Feasibility:_ An array of open-tab descriptors in state; tab bar renders buttons; active tab drives the detail pane; closeable via `x` buttons.
16. **Section Color Accents** — Each catalog section gets a distinct accent color applied to its cards and sidebar icon. _Feasibility:_ Map `sectionId -> accentColor` in data; apply via inline `style={{ '--accent': color }}` CSS variable on cards.

## Reactor Simulator Physics & Controls

1. **Point-Kinetics Core Loop** — Integrate the 6-group point-kinetics equations for neutron population and precursor concentrations each tick. _Feasibility:_ Pure TS function `stepKinetics(state, dt)` using RK4; store `n`, `C[6]` in a plain state object, driven by a `requestAnimationFrame` or fixed-timestep accumulator.
2. **Fixed-Timestep Accumulator** — Decouple physics dt from render framerate so results are frame-rate independent. _Feasibility:_ Classic accumulator pattern (`while(acc >= dt){ step(); acc -= dt }`) inside a rAF loop; dt configurable (e.g. 10 ms sim step).
3. **Six Delayed-Neutron Groups** — Model U-235 delayed neutron fractions and decay constants (β_i, λ_i) as configurable tables. _Feasibility:_ Constant arrays in a `physics/constants.ts`; effective β and generation time Λ derived at init; swappable per fuel type.
4. **Prompt-Jump Approximation Mode** — Optional stiff-solver shortcut that treats prompt neutrons as instantaneous near critical. _Feasibility:_ Branch in the integrator selecting algebraic prompt-jump vs full RK4; toggled via a `solverMode` flag for speed on slow devices.
5. **Adaptive Stiff Integrator** — Switch to an implicit/semi-implicit method when reactivity swings make ODEs stiff. _Feasibility:_ Implement backward-Euler or Rosenbrock step in TS; detect stiffness by monitoring step error and clamp dt automatically.
6. **Reactivity Aggregator** — Central function summing all reactivity contributions (rods, boron, temperature, xenon, samarium, void) into total ρ. _Feasibility:_ `computeReactivity(state)` returns pcm; each contributor is a pure function so terms are unit-testable in isolation.
7. **Doppler Fuel Temperature Feedback** — Negative reactivity from fuel temperature rise via a Doppler coefficient. _Feasibility:_ `ρ_dop = α_f * (T_fuel - T_ref)`; α_f constant in pcm/°C; T_fuel fed from thermal model.
8. **Moderator Temperature Feedback** — Coolant/moderator temperature reactivity coefficient (MTC). _Feasibility:_ `ρ_mod = α_m * (T_mod - T_ref)`; MTC sign configurable to demonstrate positive vs negative coefficient scenarios.
9. **Boron Concentration Control** — Soluble boron (ppm) adds negative reactivity; operator can borate/dilute. _Feasibility:_ State `boronPpm`, reactivity `ρ_b = -k_b * ppm`; borate/dilute controls change ppm at a bounded rate per second.
10. **Boron Dilution/Boration Rate Limits** — Model charging/letdown flow so boron changes are gradual, not instant. _Feasibility:_ Integrate `dPpm/dt` bounded by pump capacity constant; slider sets target ppm, engine ramps toward it.
11. **Xenon-135 Poisoning Model** — Track I-135 → Xe-135 buildup/decay and its large negative reactivity. _Feasibility:_ Two coupled ODEs for iodine and xenon concentrations with production/absorption terms; integrate alongside kinetics each step.
12. **Xenon Transient / Dead-Time** — Simulate post-shutdown xenon peak and the startup dead-time window. _Feasibility:_ Same Xe/I ODEs; after SCRAM, xenon rises then decays — surfaces the classic ~9h peak; deterministic given the model.
13. **Samarium-149 Poisoning** — Promethium → Samarium chain adding permanent-ish negative reactivity. _Feasibility:_ Coupled ODE pair for Pm-149/Sm-149; slower dynamics than xenon; same integrator hook.
14. **Control Rod Bank Model** — Multiple rod banks with position (0–100% withdrawn) mapped to reactivity worth curves. _Feasibility:_ Per-bank S-shaped integral worth curve (tabulated or `erf`-based); differential worth = derivative; total rod ρ summed.
15. **Integral vs Differential Rod Worth** — Compute and expose both integral worth and differential worth (pcm/step). _Feasibility:_ Numerically differentiate the tabulated integral-worth curve; used by controls and validation, pure TS.
16. **Rod Insertion/Withdrawal Speed Limits** — Rods move at a bounded steps-per-second rate, not instantly. _Feasibility:_ Target position vs actual position with rate clamp per tick; matches real rod drive mechanism speeds.
17. **Rod Overlap & Sequencing** — Bank overlap logic so banks withdraw in a programmed sequence. _Feasibility:_ Config table of bank start/stop positions; a controller maps a single "rod demand" to individual bank positions.
18. **SCRAM / Reactor Trip Logic** — Full rapid rod insertion driven by trip setpoints. _Feasibility:_ `checkTrips(state)` evaluates conditions; on trip, all banks driven to 0% at max gravity-drop speed; latched until reset.
19. **Configurable Trip Setpoints** — Editable thresholds for high power, high temp, high rate, low flow, etc. _Feasibility:_ Setpoint object with comparison operators; each evaluated per tick; UI numeric inputs bound to the object.
20. **Manual SCRAM Button** — Operator-triggered emergency shutdown with confirmation. _Feasibility:_ React button dispatches a `scram` action into the engine; big red styled control, guarded by a hold-to-confirm interaction.
21. **Reactor Period Calculation** — Compute stable/asymptotic reactor period (seconds) and doubling time. _Feasibility:_ `period = n / (dn/dt)`; derive from successive states or inhour-based estimate; display-ready number from engine.
22. **Startup Rate (DPM) Meter Feed** — Provide startup rate in decades-per-minute for source-range indication. _Feasibility:_ `SUR = 26.06 / period`; pure derived value exported from engine state each tick.
23. **Neutron Source Term** — Constant/decaying startup neutron source so power never truly hits zero. _Feasibility:_ Add source `S` term to the `n` equation; models Cf-252 or photoneutron source; prevents log-scale singularities.
24. **Subcritical Multiplication Model** — Show source multiplication M = 1/(1-k) as rods withdraw during approach-to-critical. _Feasibility:_ Compute k_eff from reactivity, derive M; supports the classic 1/M plot data (engine emits the values).
25. **Decay Heat Model** — Post-shutdown decay heat via ANS-5.1-style summation or exponential fit. _Feasibility:_ Sum of exponential terms (`Σ a_i·e^(-b_i·t)`) scaled to pre-shutdown power; pure function of shutdown time.
26. **Lumped Thermal-Hydraulic Model** — Fuel, cladding, and coolant node temperatures from a heat-balance ODE set. _Feasibility:_ 2–3 node lumped-capacitance model; `dT/dt` from power in minus heat removed; integrated with kinetics.
27. **Coolant Flow & Pump Model** — Primary coolant mass flow affects heat removal and low-flow trips. _Feasibility:_ Pump state sets flow rate; heat removal term scales with flow; RCP trip/coastdown modeled with exponential decay.
28. **Pump Coastdown Dynamics** — Reactor coolant pump flow decays exponentially after trip/power loss. _Feasibility:_ `flow = flow0·e^(-t/τ)` with inertia τ; feeds thermal model; triggers low-flow SCRAM realistically.
29. **Steam Generator / Secondary Load** — Simple secondary-side heat sink representing turbine load demand. _Feasibility:_ Load-following term removing heat proportional to a "load %" slider; couples to primary via a heat-transfer coefficient.
30. **Pressurizer Pressure Model** — Primary pressure as a function of average coolant temperature and heaters/spray. _Feasibility:_ Simplified saturation/insurge-outsurge relation; pressure state used for low/high-pressure trips.
31. **Void Reactivity (Boiling) Term** — Reactivity change from coolant voiding at high temperature/low pressure. _Feasibility:_ Void fraction estimated from margin to saturation; `ρ_void = α_v · voidFraction`; added to aggregator.
32. **Deterministic Seeded RNG** — All stochastic elements (source noise, malfunction timing) reproducible from a seed. _Feasibility:_ Small PRNG (mulberry32/xorshift) seeded once; store seed in scenario so runs replay identically.
33. **Detector Noise Injection** — Optional realistic statistical noise on neutron flux readings. _Feasibility:_ Seeded RNG adds Poisson-like jitter to the exported flux value only (not core state), toggleable for realism vs clean demos.
34. **Sim Speed Multiplier** — Run at 0.25x–100x real time for slow xenon or fast transients. _Feasibility:_ Multiply accumulator's virtual dt; physics unchanged, just more steps per frame; clamp to keep integrator stable.
35. **Pause / Resume Control** — Freeze the entire simulation without losing state. _Feasibility:_ Boolean `running` gate around the step loop; rAF keeps rendering, physics halts; trivial React state.
36. **Single-Step Debug Mode** — Advance exactly one physics step per click for teaching/debugging. _Feasibility:_ When paused, a "Step" button calls `step(dt)` once; shows before/after deltas in a debug panel.
37. **Simulation State Snapshot/Restore** — Save and reload the full engine state at any moment. _Feasibility:_ State is a serializable plain object; `JSON.stringify` to localStorage or Tauri fs; restore rehydrates the engine.
38. **Scenario Definition Format** — Declarative JSON scenarios (initial conditions + scripted events). _Feasibility:_ Typed scenario schema; loader sets initial state and queues timed events; mirrors the catalog's data-driven ethos.
39. **Scripted Event Timeline** — Time- or condition-triggered events (insert malfunction at t=120s). _Feasibility:_ Event queue sorted by trigger; each tick checks due events and applies mutations to engine state.
40. **Malfunction: Stuck Control Rod** — A rod bank refuses to move (dropped or stuck out). _Feasibility:_ Flag on a bank overrides its demanded position; reactivity reflects the frozen rod; injected via scenario or manual toggle.
41. **Malfunction: Dropped Rod Event** — Single rod drops fully in, causing flux tilt and power drop. _Feasibility:_ Force one bank to 0%, apply an asymmetric worth penalty; engine emits the transient for visualization.
42. **Malfunction: Boron Dilution Accident** — Uncontrolled dilution adds positive reactivity over time. _Feasibility:_ Scripted event ramps boronPpm down at abnormal rate; tests operator response and trip logic.
43. **Malfunction: Loss of Coolant Flow** — Trip one or all RCPs to force a flow transient. _Feasibility:_ Event sets pump state off; coastdown + reduced heat removal cascade through thermal model to trips.
44. **Malfunction: Loss of Feedwater** — Secondary heat sink degrades, raising primary temperature. _Feasibility:_ Event reduces the secondary load coefficient; primary heats up, driving temp-related trips.
45. **Malfunction: Instrument Failure** — A sensor freezes or reads offset while true state diverges. _Feasibility:_ Wrap exported readings in a per-sensor transform; failure applies stuck/biased override, true core state untouched.
46. **Startup Procedure Checklist Engine** — Guided step-by-step reactor startup with state validation. _Feasibility:_ Ordered steps each with a `precondition(state)` predicate; UI advances only when engine state satisfies it.
47. **Shutdown Procedure Sequence** — Guided controlled shutdown to hot/cold standby. _Feasibility:_ Same checklist engine with shutdown steps; validates rod insertion, boration, cooldown rate limits against state.
48. **Cooldown Rate Limiting** — Enforce/monitor °C-per-hour cooldown limits to model thermal stress rules. _Feasibility:_ Track dT/dt of coolant; flag or block actions exceeding a configurable limit; pure derived check.
49. **Shutdown Margin Calculator** — Compute available negative reactivity margin with most-reactive rod stuck out. _Feasibility:_ Sum rod worths minus the highest-worth bank; compare to required margin; pure function over rod config + boron.
50. **Critical Rod Position Predictor** — Estimate the rod height at which the core goes critical for given conditions. _Feasibility:_ Solve total reactivity = 0 for rod position via bisection over the worth curve; pure TS root-find.
51. **Inhour Equation Solver** — Convert reactivity to period (and vice versa) exactly. _Feasibility:_ Root-find the inhour equation over the 6 groups (bisection/Newton in TS); used for period validation and teaching.
52. **Reactivity Balance Ledger** — Live breakdown table of every reactivity source in pcm and their sum. _Feasibility:_ Engine already computes each term separately; expose them as a `{source: pcm}` map for a control-panel ledger.
53. **Control Input Rate Governor** — Global safeguard clamping how fast any operator input changes core state. _Feasibility:_ Wrapper on all setter actions enforcing max delta/tick; prevents non-physical instant jumps; centralizes realism.
54. **Interlock / Permissive Logic** — Block illegal actions (e.g. rod withdrawal blocked below source count). _Feasibility:_ `permissives(state)` returns which controls are enabled; UI disables buttons; engine rejects blocked actions defensively.
55. **Auto Reactor Control Mode** — Optional PID auto-controller holding target power by adjusting rods. _Feasibility:_ Standard PID in TS driving rod demand from power error; toggle between manual and auto; gains configurable.
56. **Load-Follow Program** — Auto mode tracks a power-vs-time ramp (e.g. daily load-follow curve). _Feasibility:_ Setpoint generator feeds the PID a scheduled target; scenario-defined ramp; deterministic.
57. **Trip History / Event Log** — Timestamped log of trips, alarms, and operator actions. _Feasibility:_ Append-only array of `{t, type, detail}`; engine pushes events; exportable as JSON/CSV via Tauri fs.
58. **Time-Series Data Recorder** — Ring-buffer of key variables for post-run export/replay. _Feasibility:_ Fixed-length circular buffer of samples at a chosen interval; downsampled; handed to viz agent or exported.
59. **Deterministic Replay from Log** — Re-run a session exactly from seed + input log. _Feasibility:_ Since RNG is seeded and inputs are logged with timestamps, replaying the input queue reproduces the run bit-for-bit.
60. **Physics Validation Test Suite** — Vitest tests asserting known analytic results (steady state, reactivity→period). _Feasibility:_ Pure functions make this easy; assert e.g. inhour solver vs prompt-jump vs full solver agree within tolerance.
61. **Reactivity Units Toggle** — Switch reactivity display between pcm, %Δk/k, and dollars/cents. _Feasibility:_ Pure conversion helpers using β_eff; UI unit selector; engine stores canonical pcm internally.
62. **Web Worker Physics Thread** — Run the integrator off the main thread to keep UI at 60fps. _Feasibility:_ Move engine into a Web Worker; postMessage state snapshots; works in both browser and Tauri webview.

## Reactor Visualization & Dashboards

## Performance & Optimization

1. **Route-Level Code Splitting** — Split catalog, detail, and reactor sim into lazily-loaded chunks so startup only ships the shell. _Feasibility:_ `React.lazy(() => import('./ReactorSim'))` + `<Suspense>`; Vite emits separate chunks automatically per dynamic import.
2. **Reactor Sim Lazy Bundle** — Keep the entire physics/point-kinetics module out of the initial download until the sim tab opens. _Feasibility:_ dynamic `import()` guarded behind the sim route; Vite tree-splits it and prefetches on hover via `<link rel="prefetch">`.
3. **Module List Virtualization** — Render only visible cards of the 314 modules instead of all at once. _Feasibility:_ `@tanstack/react-virtual` `useVirtualizer` over the grid container; measures rows and windows the DOM to ~20 nodes.
4. **Grid Virtualization with Dynamic Row Height** — Virtualize the section/group grid where cards vary in height. _Feasibility:_ TanStack Virtual `measureElement` callback re-measures each rendered card, supporting variable sizes without layout thrash.
5. **useTransition for Filter Updates** — Mark catalog re-filtering as a non-urgent transition so typing never blocks. _Feasibility:_ `startTransition(() => setFiltered(...))`; React 18 concurrent scheduler keeps the input paint priority high.
6. **React.memo on Module Cards** — Prevent all 314 cards re-rendering when unrelated state changes. _Feasibility:_ wrap `ModuleCard` in `React.memo` with a shallow-prop equality; cards only re-render when their own module/selected props change.
7. **Memoized Search Index** — Build the bilingual search index once, not per render. _Feasibility:_ `useMemo(() => buildIndex(modules), [modules])`; precompute lowercased EN + zh-TW token maps so per-keystroke filtering is O(n) string compares only.
8. **Precomputed Search Tokens** — Ship a build-time-generated token/index file so no client-side indexing happens at startup. _Feasibility:_ a Vite plugin or prebuild script emits `search-index.json` from the module data; app just loads it.
9. **Web Worker Sim Loop** — Run point-kinetics integration off the main thread to keep UI at 60fps. _Feasibility:_ dedicated Worker via `new Worker(new URL('./sim.worker.ts', import.meta.url), {type:'module'})`; Vite bundles worker chunks natively.
10. **Comlink Worker Bridge** — Ergonomic typed RPC to the sim worker instead of raw postMessage. _Feasibility:_ `comlink` wraps the worker in a proxy; TypeScript types flow through for `sim.step(dt)` calls.
11. **SharedArrayBuffer Sim State** — Share reactor state arrays between worker and UI with zero-copy. _Feasibility:_ `SharedArrayBuffer` + `Float64Array` view; requires COOP/COEP headers which Tauri can set on its asset server.

## Testing & QA

1. **Physics conservation invariants** — Assert reactor point-kinetics never violates neutron/precursor non-negativity across a step. _Feasibility:_ Vitest `expect(n).toBeGreaterThanOrEqual(0)` looped over every integrator output frame; pure TS physics fns are directly importable.
2. **Reactivity-to-period golden checks** — For fixed reactivity steps, assert stable reactor period matches inhour-equation analytic value. _Feasibility:_ Compute expected period from delayed-neutron params in the test, compare sim output with `toBeCloseTo(x, 4)`.
3. **fast-check reactivity fuzzing** — Property test: any bounded reactivity input keeps flux finite and non-NaN. _Feasibility:_ `fast-check` `fc.assert(fc.property(fc.double({min,max}), r => Number.isFinite(step(r).n)))`.
4. **Integrator step-size independence** — Property test that RK4 result at t converges as dt halves (Richardson-style). _Feasibility:_ fast-check generates dt pairs; assert `|y(dt) - y(dt/2)| < tol` monotonic shrink.
5. **Energy/temperature monotonicity** — Property: with positive power and no cooling, fuel temperature never decreases. _Feasibility:_ fast-check over power sequences; assert `T[i+1] >= T[i]` in Vitest.
6. **Golden-file transient snapshots** — Store reference CSV traces of canonical transients (SCRAM, rod withdrawal) and diff new runs. _Feasibility:_ Serialize trace array to JSON, `toMatchSnapshot()` or committed golden file compared with tolerance helper.
7. **Doubling-time golden test** — Assert supercritical transient hits known power-doubling time. _Feasibility:_ Run sim to 2x power, record time index, compare against committed golden value with `toBeCloseTo`.
8. **SCRAM shutdown assertion** — After control-rod insertion command, power decays below threshold within N seconds. _Feasibility:_ Vitest drives the sim step loop, asserts `power < 0.01 * initial` before deadline.
9. **Steady-state fixed point** — Zero net reactivity keeps flux constant within epsilon over long run. _Feasibility:_ Loop 10k steps, assert `Math.abs(n[end]-n[0]) < 1e-6`.
10. **Determinism / reproducibility test** — Same seed + inputs yields byte-identical trace twice. _Feasibility:_ Run sim twice, `expect(runA).toEqual(runB)`; guards against hidden Date.now/Math.random.
11. **Catalog Zod schema validation** — Validate all 314 module JSON entries against a Zod schema in a test. _Feasibility:_ `zod` `.parse()` inside `it.each(modules)`; fails CI on malformed/missing fields.
12. **Unique module ID test** — Assert no duplicate module IDs across the catalog. _Feasibility:_ Build a `Set`, `expect(set.size).toBe(modules.length)` in Vitest.
13. **Bilingual completeness test** — Every module has both `en` and `zh-Hant` name/description fields. _Feasibility:_ Iterate catalog, assert both locale keys present and non-empty strings.
14. **i18n key parity test** — EN and zh-Hant translation JSONs have identical key sets. _Feasibility:_ Recursively flatten both trees, `expect(enKeys).toEqual(zhKeys)`; catches missing translations.
15. **No orphan i18n keys** — Assert every translation key is referenced in source (or flag unused). _Feasibility:_ Grep `t('key')` usages via `fast-glob` + regex in a Vitest test, compare to key list.
16. **No hardcoded strings scan** — Test fails when JSX contains untranslated literal text. _Feasibility:_ Parse `.tsx` with `@typescript-eslint/parser` AST or regex heuristic in a test asserting no bare text nodes outside `t()`.

## Security & Privacy

1. **Strict CSP Header** — Lock the webview to a restrictive Content-Security-Policy allowing only self, no inline scripts. _Feasibility:_ Set `app.security.csp` in `tauri.conf.json` (e.g. `default-src 'self'; script-src 'self'`); Tauri v2 injects it and nonces its own IPC bootstrap.
2. **Per-Command Capability Files** — Split Tauri capabilities into narrowly-scoped JSON files granting each window only the exact commands it needs. _Feasibility:_ Tauri v2 `capabilities/*.json` with explicit `permissions` arrays and `windows` globs; the reactor window gets no `run_command` access at all.
3. **run_command Allowlist Wrapper** — Replace free-form command execution with an enum of vetted operations mapped server-side to fixed argv. _Feasibility:_ Rust `#[tauri::command]` takes an `enum Operation`, matches to a hardcoded `Command::new(...)` with fixed program path; user input never becomes the executable name.
4. **Argument-Only Parameterization** — Pass user data exclusively as separated args, never string-concatenated into a shell line. _Feasibility:_ Use `std::process::Command.args([...])` (no `sh -c`); Rust's `Command` does not invoke a shell, eliminating classic injection.
5. **PowerShell Argument Escaping** — Sanitize/escape all interpolated values before they reach run_powershell scripts. _Feasibility:_ Rust helper that rejects/escapes `` ` $ ; | & ( ) `` and validates against a per-parameter regex allowlist before building the script string.
6. **Path Canonicalization Guard** — Reject path-traversal in list_dir/get_env by canonicalizing and confining to allowed roots. _Feasibility:_ Rust `dunce::canonicalize` + `starts_with(allowed_root)` check; returns error on `..` escapes before any filesystem read.
7. **Native Operation Audit Log** — Append-only structured log of every native command with args, timestamp, and result code. _Feasibility:_ Rust `tracing` + a JSONL file writer in `app_data_dir()`; each `#[tauri::command]` logs entry/exit; frontend can read via a dedicated `read_audit_log` command.
8. **Audit Log Viewer Panel** — In-app React view listing recent native operations with filter by command/status. _Feasibility:_ React component calling `invoke('read_audit_log')`, virtualized list; degrades to a stub in browser mode where no audit exists.

11. **Command Confirmation Diff** — Show the exact resolved argv to the user before execution for destructive operations. _Feasibility:_ Rust returns a `preview` (program + args) without running; React modal renders it; a second `confirm_and_run` invoke with a one-time token executes.

49. **Signed Audit Log Chain** — Make the audit log tamper-evident with a hash chain. _Feasibility:_ Each JSONL entry includes `prev_hash = sha256(prev_line)` (Rust `sha2`); a verify command detects any insertion/edit.
50. **Update Rollback Protection** — Reject updates with a version lower than installed to block downgrade attacks. _Feasibility:_ Rust compares semver from the signed manifest against current `env!("CARGO_PKG_VERSION")`; refuses older versions.
51. **Permission Prompt for First Use** — First invocation of any native capability class prompts the user to grant it. _Feasibility:_ Frontend tracks granted classes in encrypted local store; ungranted class triggers a React consent modal before the invoke proceeds.

55. **Injection Fuzz Harness** — Automated fuzzing of command wrappers with malicious inputs in CI. _Feasibility:_ Rust `cargo-fuzz` target feeding random/edge strings into the argument-sanitization functions, asserting no shell metacharacters survive.
56. **Privacy-Safe Crash Reports** — Scrub PII/paths from any crash dump before it is written or offered for sharing. _Feasibility:_ Rust panic hook + a scrub pass replacing user paths (`C:\Users\<name>`) and identifiers before writing the dump file.

## Collaboration, Sharing, Onboarding & Help

1. **Shareable Sim Link** — Encode full reactor scenario (control-rod, coolant, params) into a URL query string for one-click sharing. _Feasibility:_ Serialize sim state to JSON, LZ-compress with `lz-string`, base64url into `?s=` param; read on mount via `URLSearchParams` and `history.replaceState` to keep URL live without a router.
2. **Catalog View Deep Link** — URL encodes current section/group/card/detail + search query so a link reopens the exact catalog spot. _Feasibility:_ Hash-based state (`window.location.hash`) parsed into React state on load; `hashchange` listener syncs both ways — no router lib needed.
3. **Export Scenario File** — "Save scenario" button downloads a `.wforge.json` file of the sim setup. _Feasibility:_ `Blob` + `URL.createObjectURL` + anchor `download` in browser; in Tauri use `@tauri-apps/plugin-dialog` save dialog + `plugin-fs` `writeTextFile` for a native Save-As.

5. **Scenario Version Guard** — Embed a schema version in exported files and migrate/warn on load. _Feasibility:_ Store `{version: 3, data}`; a small migration map of functions upgrades older payloads before hydrating state; show a toast if unmigratable.
